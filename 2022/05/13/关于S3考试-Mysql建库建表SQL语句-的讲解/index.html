    关于S3考试 Mysql建库建表SQL语句 的讲解

    本文字数： 1.7k 阅读时长 ≈ 2 分钟

    老样子,应付S3考试出的一个公式 –==赵牛逼专属==

    建库语句

    前面两个单词是固定的

    S3结业考试题没有固定库名 随便写就行

    建表语句

    前面两个单词是固定的

    表名就是题目上面给的

    例子

    基本上两个表的sql语句都是下面这种

    对于建表里面的语句

    格式

    ​ 1. 第一个是除了主键的写法

    如果写非空的那一栏不写 或者是可空==not null==就可以不写

    ​ 2. 第二个是主键 并且自动增长的写法

    数据类型

    直接来张表 基本上涉及到考试能用到的 对着写就行了

    数据类型 单词
    长整型 bigint
    整形 int
    字符串 varchar(50)
    日期类型 date

    其他应该注意的

    每一行后面最好加 ;

    在建表里面的 列什么的 每写完一个列就需要加 , 最后一个不需要加 这个必须加

    主键外键怎么建

    建完表之后 刷新显示两个表 在列多的表名上面右键

    选择 ==关联/外键==

    点击创建之后

    在弹出的窗口上面 ==引用表== 选择另一个表

    在左边的窗口选择备注带 主键或者外键的列 通常是列名中有id的

    然后点击创建 就创建好了

    新增数据

    先新增列少的那张表

    在新增列多的那张表

    不然会报错

    ==强烈建议 新增三条数据==

    两表查询

    怎么写

    打个例子举个锤子

    第一行

    select ==列少的表== . ==除了主键的那个列== , ==另一个表== .*

    第二行

    from ==列少的表== . ==主键== inner join ==列多的表== on ==列少的表== . ==主键== = ==列多的表== . ==外键==

    外键 :==通常是备注里面说对应什么什么的主键 或者是带有外键字样的 就是上面建立主外键 去选择的那个列==

    第三行

    肯定有条件

    第四行

    <if test=" ==外键== !=null and ==外键== != ‘’ and ==外键== != -1">

    上面的 使用带下拉框

    <if test=" ==外键== !=null and ==外键== != ‘’>

    不带下拉框的

    带下拉框的有一个 ==外键== != -1 的判断

    第五行

    也是分为两种情况

    带下拉框的

    ==外键== = #{==外键==}

    不带下拉框的

    ==外键== like concat(‘%’,#{ ==外键== }, ‘%’)

    第八行

    order by ==时间列== desc

    就是这个列

    修改状态

    适用于来回切换 只需要传id 就行

    如果原来的值是1 那么计算完之后就是0

    如果 值是1 那么 1-这个值就是 0 1-1=0

    如果值是0 那么 1-这个值就是 1 1-0=1
