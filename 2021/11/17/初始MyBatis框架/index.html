    初始MyBatis框架

    发表于 2021-11-17 更新于 2021-11-19 分类于 Java开发之SSM企业轻量级开发框架
    本文字数： 5.7k 阅读时长 ≈ 5 分钟

    框架的简单介绍

    定义

    是一种经过检验的,具有特定功能的未成品软件

    1. 是一个应用程序的半成品
    2. 提供可重用的公共结构
    3. 按一定规则组织的一组组件

    优点

    1. 不用再考虑公共问题
    2. 专心在业务实现上
    3. 结构统一，易于学习、维护
    4. 新手也可写出好程序

    主流框架

    MyBatis

    1. 半自动化的ORM实现
    2. DAO层 对数据库层打交道
    3. 动态SQL
    4. 小巧灵活、简单易学

    Spring

    1. 依赖注入容器 / AOP实现

    2. 声明式事务

    3. 简化Java EE应用

    4. 黏合剂，将大家组装到一起

    Spring MVC

    1. 结构最清晰的MVC Model2实现
    2. 高度可配置，支持多种视图技术
    3. 定制化开发

    Struts

    1. MVC设计模式的实现
    2. 拦截器
    3. 可变和可重用的标签

    HiberNate

    1. ORM，简化数据库操作
    2. DAO层

    持久化

    1. 定义：持久化是程序数据在瞬时状态和持久状态间转换的过程
    2. 实现持久化的方式
      1. 文件
      2. 数据库

    ORM

    1. ORM一般指对象关系映射。 对象关系映射（英语：Object Relational Mapping，简称ORM，或O/RM，或O/R mapping）
      1. ​ O Object 对象 R Relational 关系 M Mapping 映射
      2. ​ 编写程序的时候，以面向对象的方式处理数据
      3. 保存数据的时候，却以关系型数据库的方式存储
    2. ORM解决方案包含下面四个部分
      1. 在持久化对象上执行基本的增、删、改、查操作
      2. 对持久化对象提供一种查询语言或者API
      3. 对象关系映射工具
      4. 提供与事务对象交互、执行检查、延迟加载以及其他优化功能

    MyBatis简介

    1. MyBatis前身是iBatis，本是Apache的一个开源的项目
    2. 官方网站:http://mybatis.org
    3. ORM框架
    4. 特点
      1. 实体类和SQL语句之间建立映射关系
      2. 基于SQL语法，简单易学
      3. SQL语句封装在配置文件中，便于统一管理与维护，降低程序的耦合度

    上机实操

    4. 创建实体类（POJO）

    6. 定义SQL映射文件 类似于sql java代码
    7. 创建测试类
      1. 读取MyBatis核心配置文件
      2. 创建SqlSessionFactory实例
      3. 创建SqlSession实例
      4. 创建Mapper实例执行SQL映射文件中的SQL语句

    Mybatis基本使用(X)

    1. 创建web项目，加jar包
    2. 编写mybatis核心配置文件mybatis_config.xml
    3. 编写实体类(pojo)
    4. 编写接口
    5. 编写sql映射文件
    6. 创建业务类
      1. 读取MyBatis核心配置文件

    MyBatis框架优缺点

    优点

    1. 与JDBC相比，减少了50%以上的代码量
    2. 最简单的持久化框架，小巧并简单易学
    3. SQL代码从程序代码中彻底分离，可重用
    4. 提供XML标签，支持编写动态SQL
    5. 提供映射标签，支持对象与数据库的ORM字段映射

    缺点

    1. SQL语句编写工作量大，对开发人员有一定要求
    2. 数据库移植性差

    MyBatis专注于SQL本身，是一个足够灵活的DAO层解决方案，适用于性能要求较高或者需求多变的互联网项目

    MyBatis基本要素

    1. MyBatis的核心对象
      1. SqlSessionFactoryBuilder
      2. SqlSessionFactory
      3. SqlSession
    2. MyBatis核心配置文件
    3. SQL映射文件

    核心接口和类的关系

    SqlSessionFactoryBuilder

    用过即丢，其生命周期只存在于方法体内
    负责构建SqlSessionFactory，并提供多个build()方法的重载

    SqlSessionFactory

    SqlSessionFactory是每个MyBatis应用的核心
    作用：创建SqlSession实例

    生命周期与应用的生命周期相同

    单例 : 存在于整个应用运行时，并且同时只存在一个对象实例

    SqlSession

    包含了执行SQL所需的所有方法
    非线程安全，不能共享
    对应一次数据库会话，会话结束必须关闭

    一个SqlSession实例可以执行多次SQL语句

    SqlSession的两种使用方式

    1. 通过 namespace+id 字符串运行映射的SQL语句
    2. 基于Mapper接口方式执行SQL语句

    小结

    1. 非集成环境下的最佳实践
      1. SqlSessionFactoryBuilder的最佳使用范围是什么？
        1. 用过即丢
        2. 推荐作用域范围：局部变量
      2. SqlSessionFactory的最佳使用范围是什么？
        1. 生命周期与应用的生命周期相同
        2. 最佳作用域范围：应用的全局作用域
      3. SqlSession的最佳使用范围是什么？
        1. 线程级
        2. 一个request请求期间

    MyBatis核心配置文件

    主要元素

    configuration 根节点
    properties 描述一些外部的、可替代的属性
    settings 修改MyBatis的运行时行为
    typeAliases 为Java类型指定一个别名
    typeHandlers 类型处理器
    objectFactory 对象工厂
    plugins 插件
    environments 环境配置
    environment 一组环境配置
    transactionManager 事务管理器
    dataSource 数据源
    databaseIdProvider 启用多数据库厂商支持
    mappers 映射器

    配置properties元素

    配置properties元素的两种方式

    通过指定外部属性文件

    如：database.properties

    直接配置property子节点

    若两种方式同时都用了，哪种方式生效？

    resource属性值会覆盖property子节点配置的同名属性值

    settings元素

    用来修改MyBatis运行时的行为方式

    主要是MyBatis的一些全局配置属性的设置

    设置项 描述 允许值 默认值
    cacheEnabled 对在此配置文件下的所有cache 进行全局性开/关设置 true | false true
    lazyLoadingEnabled 全局性设置懒加载。如果设为false，则所有相关联的都会被初始化加载 true | false true
    autoMappingBehavior MyBatis自动映射匹配级别 NONE | PARTIAL | FULL PARTIAL
    logImpl 指定MyBatis应使用的日志记录实现。如果不设置将自动发现日志记录实现 SLF4J | LOG4J | LOG4J2 | …… 无

    typeAliases元素

    类型别名
    在SQL映射文件中简化冗长的Java类名

    environments元素

    1. environments元素
      1. 可以通过environment子元素节点配置MyBatis的多套运行环境
      2. 必须通过default属性指定其中一个为默认运行环境
    1. 事务管理器transactionManager

    1. dataSource

      1. dataSource元素使用基本的JDBC数据源接口来配置JDBC连接对象的资源
      2. 有三种内建的数据源类型

    mappers元素

    SQL映射文件定义SQL语句映射
    须在配置中引用SQL映射文件

    方式一：使用类资源路径获取资源

    方式二：使用Mapper接口的包路径
